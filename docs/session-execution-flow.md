# 1on1セッション実行フロー（確定版）

## 概要

Fit Gateで「Session解放」判定を受けたユーザーに対して、1on1テストセッションを実施するための実行フローです。α期間中は手動運用を前提とし、まずは**3件のセッション完了**を目標とします。

---

## 実行フロー全体図

```
fitgate_result_session 発火
  ↓
【ステップ1】申込確認メール送信（手動）
  ↓
【ステップ2】日程調整（カレンダーまたは手動メール）
  ↓
【ステップ3】1on1セッション実施（90分）
  ↓
【ステップ4】世界線メモ作成
  ↓
【ステップ5】sessionDone フラグ記録
  ↓
【ステップ6】Evidence Pack 用に保存（3件溜まったタイミング）
```

---

## ステップ1: 申込確認メール送信（手動）

### トリガー

- `fitgate_result_session` イベントが発火（Fit Gateで「Session解放」判定）
- または、`testSessions` テーブルに新しい申込が追加

### 実施者

- PdM または LP Team

### 手順

1. **データベースで申込内容を確認**

   **方法A: Manus Management UI（推奨）**
   - Management UI → Database パネルを開く
   - `fitGateResponses` テーブルを選択
   - `judgmentResult = 'session'` でフィルタリング
   - 最新の申込を確認

   **方法B: SQL直接クエリ**
   ```sql
   SELECT * FROM fitGateResponses
   WHERE judgmentResult = 'session'
   AND sessionDone = false
   ORDER BY createdAt DESC;
   ```

2. **申込確認メールを作成**

   - `docs/session-confirmation-email-template.md` を開く
   - テンプレートをコピー
   - `{変数}` 部分を実際の値に置き換え
     - `{お名前}` → `fitGateResponses.email` から推測（または手動入力）
     - `{メールアドレス}` → `fitGateResponses.email`
     - `{現在の住宅状況}` → `fitGateResponses.q2HousingStatus`
     - `{世帯年収レンジ}` → `fitGateResponses.q4IncomeRange`
     - `{物件価格帯}` → `fitGateResponses.q3PriceRange`
     - `{ゴールモード}` → `fitGateResponses.q10PreferredApproach`
     - `{希望日時}` → 手動入力（または「ご都合の良い日時をお知らせください」）
     - `{その他ご要望}` → `fitGateResponses.q9CurrentQuestion`

3. **メール送信**

   - Gmail / Outlook などから手動送信
   - 件名：`[Exit Readiness OS] 1on1テストセッションのお申込みありがとうございます`
   - 送信先：`fitGateResponses.email`

### チェックリスト

- [ ] データベースで申込内容を確認
- [ ] テンプレートの `{変数}` をすべて置き換え
- [ ] メール送信完了
- [ ] 送信記録をスプレッドシートまたはNotionに記録

## ステップ2: 日程調整（カレンダーまたは手動メール）

### 方法A: Calendly（推奨）

1. **Calendlyで予約枠を設定**
   - イベントタイプ：1on1テストセッション
   - 所要時間：90分
   - 利用可能時間：平日夜（19:00〜21:00）、週末（10:00〜18:00）

2. **ユーザーに予約リンクを送信**
   - 申込確認メールに記載済み
   - ユーザーが自分で予約

3. **予約確定後、Zoom リンクを自動送信**
   - Calendly の自動メール機能を使用

### 方法B: 手動メール

1. **ユーザーから候補日時を受信**
   - 申込確認メールへの返信で候補日時を受け取る

2. **日程確定メールを送信**
   ```
   件名：[Exit Readiness OS] 1on1セッション日程確定のお知らせ

   {お名前} 様

   1on1テストセッションの日程が確定しました。

   日時：{確定日時}（例：1月15日（月）19:00〜20:30）
   所要時間：90分
   実施方法：Zoom

   Zoomリンク：{Zoomリンク}

   当日は以下の情報をご用意ください：
   - 世帯年収（手取りまたは額面）
   - 現在の金融資産（概算でOK）
   - 住宅ローン残高（持ち家の場合）
   - 月々の生活費（ざっくりでOK）
   - 比較したい世界線（例：賃貸継続 vs 6,000万購入 vs 8,000万購入）

   それでは、当日お会いできることを楽しみにしております。

   Exit Readiness OS チーム
   ```

### チェックリスト

- [ ] 日程確定
- [ ] Zoom リンク作成
- [ ] 日程確定メール送信
- [ ] カレンダーに予定追加

## ステップ3: 1on1セッション実施（90分）

### 事前準備

- [ ] Zoom リンク動作確認
- [ ] `docs/session-worldline-memo-template.md` を開いてメモ準備
- [ ] スプレッドシート（世界線比較計算用）を準備

### セッション構成（90分）

#### 1. 前提の整理（15分）

- 世帯構成（年齢、家族構成、居住地）
- 収入・資産（世帯年収、金融資産、住宅ローン残高）
- キャリア・ライフイベント（海外駐在、転職、子どもの進学など）
- 現在の住宅状況（賃貸 or 持ち家、月額費用）
- 検討中の選択肢（6,000万購入、8,000万購入など）

**メモ:** `docs/session-worldline-memo-template.md` の「1. 前提の整理」セクションに記録

#### 2. 世界線の設定（15分）

- 比較したい選択肢を2〜3本に絞る
- 各世界線の前提を明確化
  - 住宅（賃貸継続 / 購入）
  - 資産運用（年利3%など）
  - キャリア（現職継続 / 転職 / 海外駐在）
  - ライフイベント（子どもの進学、親の介護など）

**メモ:** `docs/session-worldline-memo-template.md` の「2. 比較した世界線」セクションに記録

#### 3. 指標の計算（40分）

- スプレッドシートで以下の指標を計算
  - **40〜50代のキャッシュフロー余裕**（月あたり）
  - **60歳時点の資産残高**
  - **取り崩し開始年齢**

- 各世界線で計算し、比較表を作成

**メモ:** `docs/session-worldline-memo-template.md` の「3. 指標差分」セクションに記録

#### 4. インサイトの共有（15分）

- 数字だけでなく、「この世界線を選ぶとどうなるか」を一緒に考える
- ユーザーの価値観（余裕重視 / 資産重視 / 柔軟性重視）を確認
- 1〜3個のインサイトにまとめる

**メモ:** `docs/session-worldline-memo-template.md` の「4. インサイト」セクションに記録

#### 5. 次のステップ（5分）

- セッション後に送付する「世界線メモ」の説明
- 追加の質問があれば受け付け

### チェックリスト

- [ ] 前提の整理完了
- [ ] 世界線の設定完了
- [ ] 指標の計算完了
- [ ] インサイトの共有完了
- [ ] セッション中のメモを `docs/session-worldline-memo-template.md` に記録

2. **世界線メモを完成**

   - 比較した世界線

## ステップ5: sessionDone フラグ記録

### 実施タイミング

- 世界線メモ送付後

### 実施者

- Dev Team または LP Team

### 手順

1. **データベースで該当レコードを特定**

   ```sql
   SELECT * FROM fitGateResponses
   WHERE email = '{ユーザーのメールアドレス}'
   AND judgmentResult = 'session'
   AND sessionDone = false
   ORDER BY createdAt DESC
   LIMIT 1;
   ```

2. **sessionDone フラグを true に更新**

   **方法A: Manus Management UI（推奨）**
   - Management UI → Database パネルを開く
   - `fitGateResponses` テーブルを選択
   - 該当レコードを見つける
   - `sessionDone` カラムを `true` に変更

   **方法B: SQL直接更新**
   ```sql
   UPDATE fitGateResponses
   SET sessionDone = true
   WHERE id = {該当レコードのID};
   ```

### チェックリスト

- [ ] 該当レコードを特定
- [ ] `sessionDone` フラグを `true` に更新
- [ ] 更新完了を確認

## ステップ6: Evidence Pack 用に保存（3件溜まったタイミング）

### 実施タイミング

- セッション3件完了後

### 実施者

- PdM または LP Team

### 手順

1. **3件の世界線メモを確認**
   - セッション1件目、2件目、3件目の世界線メモを開く
   - 各ケースが異なるパターン（住宅レンジ・キャリア・イベント）を示しているか確認

2. **Evidence Pack 用にケース例を作成**
   - `docs/evidence-pack-operation-rules.md` を参照
   - 各ケース（Case A / B / C）に以下を記載
     - 匿名化した前提（年収レンジ／家族構成／住宅レンジ／海外・仕事・イベントの前提）

3. **LP v5 のトップページに反映**

   - Case A / B / C のカード内容を差し替え

4. **プレビュー環境で確認**
   - Management UI → Preview パネルで表示確認
   - PdMが最終確認

5. **本番デプロイ**

   - Management UI → Publish ボタンで本番反映

## トラブルシューティング

### ユーザーから返信がない場合

1. **リマインドメール送信**（申込確認メール送信から3日後）
   ```
   件名：[Exit Readiness OS] 1on1セッション日程調整のリマインド

   {お名前} 様

   先日お送りした1on1テストセッションの申込確認メールについて、
   ご返信をいただけていないようでしたので、念のためリマインドさせていただきます。

   ご都合の良い日時を3つほどお知らせいただけますと幸いです。

   もし既にご返信いただいている場合は、このメールは無視してください。

   Exit Readiness OS チーム
   ```

2. **1週間経っても返信がない場合**
   - セッション申込を保留扱いにする
   - `sessionDone` フラグは `false` のまま維持

### セッション中に計算が間に合わない場合

1. **セッション後に追加計算**
   - セッション中は前提の整理と世界線の設定まで完了
   - 指標の計算はセッション後に実施
   - 世界線メモ送付時に計算結果を含める

2. **次回セッションから改善**
   - 事前にスプレッドシートのテンプレートを準備
   - セッション中に入力するだけで計算が完了するようにする
