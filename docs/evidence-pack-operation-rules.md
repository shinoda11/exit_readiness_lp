# Evidence Pack 運用ルール

## 概要

LP v5のトップページに設置した「Evidence Pack（匿名ケース例）」セクションの運用ルールを定義します。このセクションは、1on1テストセッションの実際のアウトプット例を匿名化して掲載し、5万円の価値を非同期で伝えることを目的としています。

---

## セクションの目的

### 1. 価値の可視化

1on1テストセッションで提供される「世界線比較」「指標差分」「インサイト」の具体例を示すことで、ユーザーに「5万円を払う価値がある」と納得してもらう。

### 2. 信頼の担保

実際のセッション事例（匿名化）を掲載することで、「このサービスは本当に機能する」という証拠を提示する。

### 3. 適合チェックへの誘導

Evidence Packを見たユーザーが「自分もこういう比較をしてみたい」と思い、適合チェックに進むモチベーションを高める。

---

## 現在の実装状態

### セクション構成

**場所:** トップページ（Home.tsx）の Scenario（S4）セクションと AS-IS（S5）セクションの間

**見出し:** Evidence Pack（匿名ケース例）

**説明文:**
> ここに1on1の匿名アウトプット例を載せる予定です。  
> 実際のセッションが3〜5件溜まったタイミングで、具体的なケース例を追加します。

**カード構成:** 3つの空カード（Case A / Case B / Case C）

```tsx
<Card className="p-8 bg-card">
  <div className="h-48 bg-secondary/30 rounded-lg flex items-center justify-center mb-4">
    <p className="text-muted-foreground text-sm">Case A</p>
  </div>
  <h3 className="text-lg font-bold mb-2">ケース例 A</h3>
  <p className="text-sm text-muted-foreground">準備中</p>
</Card>
```

## 運用フロー

### ステップ1: データ収集条件

**条件:**
- Fit Gate で「Session解放」判定を受けたユーザー
- 実際に1on1テストセッション（90分）を実施
- セッション数が **3件** 溜まったタイミング

**データソース:**
- `fitGateResponses` テーブル（`judgmentResult = 'session'`）
- `testSessions` テーブル（実際の申込データ）
- 1on1セッション後の「世界線メモ」（後述）

### ステップ2: ケース例の作成

各ケース（Case A / B / C）に以下の要素を含める：

#### 2-1. 匿名化した前提

**含めるべき情報:**
- 世帯年収レンジ（例：年収1,800万円 → 「年収1,500〜2,000万円」）
- 子ども有無（例：「DINK」「子ども1人」）
- 住宅レンジ（例：「6,000〜8,000万円クラスのマンション検討中」）
- 海外・仕事・イベントの前提（例：「海外駐在の可能性あり」「転職検討中」）

**匿名化の方針:**
- 具体的な年齢・勤務先・物件名は記載しない
- レンジ表記を使用（例：「30代後半」「年収1,500〜2,000万円」）
- 複数のケースを組み合わせて特定されないようにする

#### 2-2. 比較した世界線

**例:**
- 世界線A: 賃貸継続（月20万円）
- 世界線B: 6,000万円マンション購入（頭金1,500万円、ローン4,500万円）
- 世界線C: 8,000万円マンション購入 + 海外駐在（賃貸に出す）

**記載形式:**
```
比較世界線：
- A: 賃貸継続（月20万円）
- B: 6,000万購入（頭金1,500万、ローン4,500万）
- C: 8,000万購入 + 海外駐在（賃貸に出す）
```

#### 2-3. 指標差分

**主要指標（3つ）:**
1. **40〜50代のキャッシュフロー余裕** - 月あたりの余裕額
2. **60歳時点の資産残高** - 金融資産の見込み額
3. **取り崩し開始年齢** - 資産を取り崩し始める年齢

**記載例:**
```
指標差分：
- 40〜50代CF余裕: A +15万/月、B +8万/月、C +12万/月
- 60歳資産: A 8,500万、B 6,200万、C 7,800万
- 取り崩し開始: A 68歳、B 72歳、C 70歳
```

#### 2-4. 1行インサイト

**書き方:**
- 「このケースでは〜」で始まる
- 判断のポイントを1文で要約
- ユーザーが「なるほど」と思える結論

**例:**
```
インサイト：
このケースでは、海外駐在の可能性を考慮すると、8,000万購入＋賃貸運用（世界線C）が60歳資産と柔軟性のバランスが最も良い。
```

### ステップ3: コンテンツ作成プロセス

1. **1on1セッション実施** - PdMがユーザーと90分のセッションを実施
2. **世界線メモ作成** - セッション後、「世界線メモ」フォーマット（後述）に沿って記録
3. **匿名化処理** - 個人情報を削除し、レンジ表記に変換
4. **ケース例ドラフト作成** - 上記の4要素（前提・世界線・指標差分・インサイト）を整理
5. **PdMとすり合わせ** - ドラフトをPdMに確認してもらい、必要に応じて修正
6. **実装** - `Home.tsx` の該当箇所を編集してケース例を差し替え

## 差し替えタイミング

### 条件

- 1on1テストセッションが **3件** 実施済み
- 3件のうち、匿名化可能なケースが **3件** ある
- 各ケースが異なるパターン（住宅レンジ・キャリア・イベント）を示している

### 実施者

- **Dev Team** - `Home.tsx` の編集とデプロイ
- **PdM** - ケース例のドラフト作成と最終確認

### 承認フロー

1. PdMが3つのケース例ドラフトを作成
2. Dev Teamに共有（Notion / Slack / GitHub Issue）
3. Dev Teamが `Home.tsx` を編集
4. PdMがプレビュー環境で確認
5. 問題なければ本番デプロイ

## 注意事項

### 1. プライバシー保護

- **個人を特定できる情報は絶対に記載しない**
  - 具体的な年齢（例：「35歳」→「30代後半」）
  - 勤務先名（例：「外資系IT企業」→「IT業界」）
  - 物件名・エリア名（例：「港区〇〇マンション」→「都内マンション」）
- 複数のケースを組み合わせて、特定されないようにする

### 2. 数字の正確性

- 指標差分の数字は実際のセッションで計算したものを使用
- 概算であることを明記（「概算」「見込み」などの表現を使う）

### 3. 更新頻度

- 最初の3件掲載後、さらに5〜10件溜まったタイミングで追加または差し替え
- 古いケース例は削除せず、別ページ（「ケース例一覧」など）にアーカイブ
